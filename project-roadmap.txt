MOUSE/TOUCH ACTIONS
- left mouse click - tap => opening context sidebar with action icons
- double mouse click - double tap => fullscreen enter/exit
- right mouse click - long tap => context menu open
- mouse wheel => zoom in/out
- mouse over locked feature (alias cursor icon)
- mouse over unlocked feature (grab cursor icon)
- left mouse click & drag on map (all-scroll cursor icon) - touch drag => map drag
- left mouse click & drag on feature dragging option (grabbing cursor icon) - touch drag => feature drag
- right mouse click & drag (croosshair cursor icon) => map rotation and tilt camera orbiting
- mouse wheel click & drag (croosshair cursor icon) => map rotation and tilt camera panning
- two fingers twisting => map rotation
- two fingers tilt => map tilt

TERMINOLOGY
- Features - set of markers and paths on a map
- Marker - visual indicator placed at a specific geographic coordinate on a map to highlight a location
- Sights - a marker attached to a path
- Path - any sequence of connected points on a map
- Circuit path - path that starts and ends at the same point
- Route - specific path to get from point A to point B by Foot/Bike/Car
- Circuit Route - a specific path to get from point A to point B and back to point A by Foot/Bike/Car
- Track - a recorded path of GPS points over time

IMPORT/EXPORT FEATURES
- import map features (support: RontoJSON, GeoJSON, GPX, KML, FIT)
- export map features (support: RontoJSON, GeoJSON, GPX, KML)
- option for import and export map features is in main context menu
- export marker/sight (support: JSON, GeoJSON, GPX, KML)
- export path/track (support: JSON, GeoJSON, GPX, KML, FIT)
1) RontoJSON (.rontojson)
- format like in Firestore database
2) GeoJSON (.geojson)
- marker - Feature with Point geometry
- path - Path (Feature with LineString geometry)
- sight - link them through custom properties like {"pathId": "route-1"}
3) GPX (.gpx)
- marker - Waypoint (<wpt>)
- path - Route (<rte, <rtept>), Track (<trk>, <trkseg>, <trkpt>)
- sight - use naming conventions
4) KML (.kml)
- marker - <Placemark> with a <Point> geometry
- path - <Placemark> with a <LineString>)
- sight - using <MultiGeometry> to link a point and a line inside one Placemark
5) FIT (.fit)
- path - track (a series of records with lat/lon, timestamp, heart rate, speed, cadence, power)
